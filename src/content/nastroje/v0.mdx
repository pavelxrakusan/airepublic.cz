## Figma → kód, ale s AI

v0 je jako mít junior frontend developera, co pracuje neuvěřitelně rychle. Popíšeš co chceš — nebo mu pošleš screenshot — a dostaneš funkční React komponentu.

## Jak ho používám

### Rychlé prototypy

Klient chce vidět návrh pricing tabulky. Místo 2 hodin v Figma:
1. Napíšu v0: "pricing table, 3 tiers, monthly/annual toggle, dark theme"
2. Za 30 sekund mám funkční komponentu
3. Pošlu klientovi preview link
4. Iteruji podle feedbacku

### Komponenty pro airepublic.cz

Některé komponenty na tomhle webu jsem prototypoval v v0 a pak přenesl do projektu. Card layouty, hero sekce, navigace — v0 je skvělý na starting point.

### Design → kód

Mám screenshot z Dribbble nebo Figma mockup? Hodím ho do v0 a dostanu implementaci v shadcn/ui + Tailwind. Není to pixel-perfect, ale je to 80 % cesty za 1 % času.

## v0 vs Claude Code

Jiný nástroj, jiný účel:

| | v0 | Claude Code |
|---|---|---|
| Zaměření | UI komponenty | Celý projekt |
| Input | Prompt + vizuály | Prompt + codebase kontext |
| Output | Izolované komponenty | Multi-file changes |
| Iterace | V prohlížeči | V terminálu |
| Backend | Ne | Ano |

Používám oboje: v0 na rychlé UI prototypy, Claude Code na implementaci do projektu.

## Verdikt

v0 je nejlepší nástroj pro rychlé generování frontend komponent. Pokud stavíš v React/Next.js se shadcn/ui, je to must-have. Pro full-stack vývoj potřebuješ víc (Claude Code, Cursor), ale v0 je výborný startovní bod.
